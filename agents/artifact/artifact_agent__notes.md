\# Artifact Agent — Notes



\## Current Intent

This agent will likely be used in its own dedicated conversation and should be treated as the owner of the analysis workspace.



\## Important Nuances

\- Artifact is meant to be more than a chart popup

\- The user wants both power and controlled reuse

\- amCharts is explicitly desired in addition to the previously selected libraries

\- Artifact should support both charting and analytical visuals

\- The user wants the AI behind Artifact to improve over time through persistence and memory



\## Memory Direction

Artifact should eventually support:

\- preference memory

\- correction memory

\- session memory

\- trade/context recall

\- consistent long-term behavior



\## Reminder

The user prefers:

\- directness

\- scope discipline

\- full files

\- no fluff

\- strong control over architecture but freedom inside the surface



\## Safe Starting Instruction

Best first instruction to the Artifact Agent:

“Start by building the local Artifact component library and registry first. Do not expand the workspace logic until the component foundation is clean.”
