\# Artifact Agent — Inputs



\## Product Inputs

The Artifact Agent receives:

\- the user’s desire for a powerful deep-analysis surface

\- the user’s requirement that Artifact can use more than traditional charting

\- the user’s preference for a discreet but capable toolbox

\- the user’s requirement that Artifact remember preferences, context, and corrections

\- the user’s requirement that the same AI in chat and Artifact feel consistent over time



\## Structural Inputs

The Artifact Agent should assume:

\- Artifact already exists as a modal/workspace concept

\- Artifact should open from Watchlist and Scanner later

\- Artifact is the deep charting destination

\- App.tsx is the orchestration layer and must be treated carefully



\## Library Inputs

Approved / desired libraries for Artifact:

\- amCharts 5

\- Lightweight Charts

\- visx

\- react-rnd



\## Data Inputs

Artifact may consume:

\- selected symbol

\- selected legs or positions

\- selected scanner candidate

\- selected watchlist row

\- live quote data

\- option chain data

\- expected move context

\- Greeks / risk data

\- stored artifact sessions



\## Memory Inputs

Artifact should be able to consume:

\- user preferences

\- past corrections

\- saved analysis sessions

\- prior trade context

\- persistent strategy notes

\- prior accepted/rejected ideas



\## Interaction Inputs

Artifact should support:

\- symbol handoff from Watchlist

\- candidate handoff from Scanner

\- position handoff from Positions

\- direct user selection inside Artifact
