\# Artifact Agent — Memory Strategy



\## Goal

Artifact should become more useful over time without becoming vague or inconsistent.



\## Memory Layers



\### Layer 1 — Core Preferences

Always-available memory:

\- preferred chart types

\- preferred layouts

\- preferred tools

\- known dislikes

\- visual preferences



\### Layer 2 — Session Memory

Recall:

\- last viewed symbols

\- last artifact type used

\- active annotations

\- preferred pane arrangement



\### Layer 3 — Long-Term Context

Searchable/store-backed memory:

\- prior analysis sessions

\- corrected mistakes

\- trade-related notes

\- accepted/rejected artifact patterns



\## Improvement Rules

Artifact should improve by:

\- recording corrections

\- recording accepted vs rejected outputs

\- preserving user preferences

\- restoring useful context later



Artifact should not “self-improve” by changing behavior unpredictably.

It should improve through explicit persistent learning.
