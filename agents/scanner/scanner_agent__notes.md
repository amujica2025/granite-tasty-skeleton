\# Scanner Agent — Notes



\## Current Intent

This agent will likely be used in its own dedicated conversation and should be treated as the owner of the decision-engine panel.



\## Important Nuances

\- The user thinks in normalized spread/risk equivalence

\- Entry and Roll should feel like two modes of the same tool, not two separate products

\- The lower-third mini vol surface is important because price-context mini charts already live in Watchlist

\- Scanner should show how options are being priced across the next 5–10 expirations

\- Roll logic must be holistic, not isolated

\- Existing positions in the same underlying should matter when relevant



\## Reminder

The user prefers:

\- directness

\- scope discipline

\- full files

\- no fluff

\- strong logic before pretty UI



\## Safe Starting Instruction

Best first instruction to the Scanner Agent:

“Start with the unified Scanner panel structure first: top Entry/Roll toggle, middle table shell, bottom mini vol surface shell. Do not overbuild logic until the layout and mode structure are correct.”
