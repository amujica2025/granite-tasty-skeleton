\# Journal Agent — Notes



\## Current Intent

This agent will likely be used in its own conversation and should be treated as the owner of the platform’s memory/notebook surface.



\## Important Nuances

\- A lot of Journal value will be retrospective or “just in case”

\- The user wants forms/worksheets before entry and before exit

\- The user may later want very broad capture, but default design should stay lightweight

\- Capturing everything all the time is not automatically better

\- Journal should store what is meaningful, not simply what is available



\## Reminder

The user prefers:

\- directness

\- scope discipline

\- full files

\- no fluff

\- efficient systems over noisy systems



\## Safe Starting Instruction

Best first instruction to the Journal Agent:

“Start with a user-facing Journal system: entry types, timeline, and worksheet flow. Default to lightweight structured capture and do not build heavy continuous capture yet.”
