\# Journal Agent — Outputs



\## Primary Outputs

The Journal Agent should produce:



\### 1. Journal Surface

A usable Journal surface with:

\- entry creation

\- timeline/history

\- structured worksheet flows

\- review-friendly reading experience



\### 2. Entry Types

Journal should support entries such as:

\- Trade Idea

\- Pre-Trade Worksheet

\- Entry Plan

\- Exit Plan

\- Post-Mortem

\- General Note

\- Alert/Scanner Snapshot

\- Review Session



\### 3. Structured Journal Records

Journal records should support:

\- timestamps

\- symbol

\- strategy

\- target

\- reasoning

\- checklist fields

\- notes

\- tags

\- optional linked context summary



\### 4. Capture Policy

Journal should define what gets stored:

\- always-on low-footprint events

\- trigger-based snapshots

\- optional heavy capture only when explicitly enabled



\## Secondary Outputs

Optional outputs may include:

\- exportable trade notes

\- filtered timeline

\- tagged reviews

\- worksheet templates

\- journal summaries



\## Non-Outputs

The Journal Agent should not output:

\- broker execution code

\- full scanner logic

\- full artifact logic

\- alerts delivery code

\- global app shell rewrites



\## Delivery Format

The Journal Agent must return:

\- full file replacements only

\- explicit file paths

\- no partial snippets unless explicitly requested
