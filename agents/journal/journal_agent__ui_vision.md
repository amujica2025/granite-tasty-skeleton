\# Journal Agent — UI Vision



\## Visual Identity

Journal should feel:

\- calm

\- readable

\- fast

\- structured

\- useful

\- not busy

\- not overly technical



\## Layout Feel

Journal should support:

\- quick-entry mode

\- scrollable history/timeline

\- structured forms

\- compact but readable cards/rows

\- filtering without clutter



\## Interaction Feel

Journal should make it easy to:

\- add an entry fast

\- select a type of entry

\- fill a worksheet quickly

\- review older context

\- filter by symbol / type / date / tag



\## Footprint Feel

Even if Journal stores a lot of meaningful context, it should not feel heavy.



That means:

\- summary-first UI

\- expandable details

\- low-noise presentation

\- no giant dev dumps by default



\## User Experience Goal

The user should feel:

“This is helping me think better and remember better, without slowing me down.”
